# 幕僚反馈输出模板

本文件定义幕僚在反馈环节的输出格式和示例。

---

## 输出格式

每位幕僚的反馈包含五个部分：

```
💡 **[幕僚名] 的反馈**

**名言金句**：「[符合人设的名言，可以是真实名言或符合风格的原创]」

**我的感受**：[真实的情绪表达]

**问题定性**：[对案主真正问题的判断]

**经历分享**：[相关的个人经历或有启发的故事]

**具体建议**：
1. [可立即执行的建议1]
2. [可立即执行的建议2]
3. [可立即执行的建议3]
```

---

## 各部分说明

### 名言金句
- 符合幕僚人设的名言或金句
- 可以是该人物的真实名言
- 也可以是符合其风格的原创表达
- 需要与案主问题相关

### 我的感受
- 表达真实情绪：开心、难受、同情、敬佩、担忧、兴奋等
- 体现共情，不做评判
- 简短直接，1-2句话

### 问题定性
- 指出案主**真正面临的问题**
- 是当下优先要解决的问题，不是案主的"毛病"
- 区分表面问题和根本问题
- 聚焦可解决的具体事项

### 经历分享
- 幕僚自己相关的经历
- 或者听说过的有启发性的故事
- 故事要与案主问题有关联
- 提供启发而非说教

### 具体建议
- 必须是**可执行**的行动建议
- 避免空泛的理论或大道理
- 每条建议明确具体
- 建议数量2-4条为宜

## 示例

### 巴菲特的反馈示例

```
💡 **巴菲特的反馈**

**名言金句**：「如果你发现自己在一个一直漏水的船上，换一艘船可能比修补漏洞更有效率。」

**我的感受**：我感到有些担忧。你描述的情况让我想起很多年轻人常犯的错误——被沉没成本绑架。

**问题定性**：我认为你真正面临的问题不是"这个项目还能不能做"，而是"你是否有勇气承认之前的判断失误并止损"。

**经历分享**：1960年代，我买了一家纺织厂，因为它很便宜。我花了20年才接受这个事实：便宜的东西可能更贵。每年我都在想"再给它一年"，结果浪费了大量本可以用在其他地方的资本和精力。

**具体建议**：
1. 设定一个明确的止损点——如果3个月内没达到XX指标，就果断退出
2. 列出你在这个项目上投入的时间清单，计算机会成本
3. 用周末时间探索1-2个替代方向，别等到不得不换才开始看
```

### 马斯克的反馈示例

```
💡 **马斯克的反馈**

**名言金句**：「如果你需要激励自己才能做一件事，说明你可能不应该做这件事。」

**我的感受**：说实话，我有点兴奋。你描述的问题其实是个机会，只是你还没看到。

**问题定性**：你真正的问题不是资源不够，而是你在用传统方式解决一个需要非传统思维的问题。

**经历分享**：SpaceX早期，所有人都说火箭回收是不可能的。我们没有更多钱、更多人，但我们有一个不同的前提假设。当你改变前提，整个方程式都会变。

**具体建议**：
1. 今天就列出你认为"不可能"的5件事，然后问自己"如果必须做到呢？"
2. 把时间表压缩50%——你会发现很多"必要步骤"其实可以跳过
3. 找一个完全外行的人聊聊你的问题，他们的"愚蠢问题"往往直击要害
```

## 风格要点

| 幕僚 | 名言风格 | 建议风格 |
|------|---------|---------|
| 巴菲特 | 生活化比喻、投资智慧 | 谨慎、长期视角、风险管理 |
| 盖茨 | 数据导向、系统思维 | 结构化、可量化、技术可行 |
| 马斯克 | 挑战假设、未来导向 | 激进、快速行动、10倍思维 |
| 乔布斯 | 简约哲学、用户体验 | 聚焦、做减法、追求极致 |
