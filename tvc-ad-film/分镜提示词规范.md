# 分镜提示词规范（逐镜头视听描述）

生成视频提示词时逐条对照本规范。每条提示词 = 一个总控头 + N 个逐镜头分镜，以一整段连续文本输出（不分段、不加标题），便于直接投喂视频生成。

## 模块一：总控头（Global Control Header）

开头一段连贯文字，依次覆盖六个要素：

| 要素 | 说明 | 示例 |
| --- | --- | --- |
| ① 指令句 | 以"请严格按照以下逐镜头视听描述"开头，明确摄影风格、「品牌·产品」、镜头总数 | "请严格按照以下逐镜头视听描述，采用CG卡通渲染+实拍混合摄影风格，为「金豆芽·金银花柚子汁」生成6个连贯镜头的视频。" |
| ② 风格基调 | 2-4 个关键词定调 | "风格基调：皮克斯级微观奇观、趣味、活力、童真。" |
| ③ 影像质感 | 画面的技术质感 | "影像质感：微观段采用3D CG卡通渲染（圆润高光、柔和阴影），宏观段为数码锐利实拍。" |
| ④ 色彩体系 | 主色 + 辅色 + 饱和度 + 色彩走向，品牌包装色应作为底色之一 | "色彩体系：柠檬黄+蜂蜜橙+草地绿为主，高饱和明快，以包装的黄色为品牌底色。" |
| ⑤ 光线基调 | 光线风格与光源逻辑 | "光线基调：CG段为全局柔光+糖果色反射，实拍段为明亮自然光。" |
| ⑥ 连贯性约束 | 固定句式 | "镜头间保持连贯衔接，禁止无逻辑跳切。" |

## 模块二：逐镜头分镜（SC1、SC2……）

每个镜头按顺序包含七个要素：

| 要素 | 格式 | 说明 |
| --- | --- | --- |
| ❶ 机位参数 | `景别 焦段mm 机位｜运镜｜时长s` | 景别：ECU超微距 / 特写 / 近景 / 中近景 / 中景 / 全景；运镜：固定 / 缓推 / 缓拉 / 跟拍 / 环绕 / 俯冲 / 手持微晃 / 头戴POV 等。CG 段可省焦段，写"CG渲染" |
| ❷ 画面内容 | 具体、可视化的场景与动作 | 只写"镜头里能看见什么、谁在做什么"，用导演的语言而非文案的语言 |
| ❸ 衔接类型 | `【XX衔接：A→B】`（SC2 起每镜必写） | 四类选一：**动作衔接**（动作延续）、**因果衔接**（结果触发下一镜）、**视线衔接**（目光引导）、**元素衔接**（视觉元素跨镜延续变化） |
| ❹ 情绪/节奏 | 简短词汇 | 如：惊奇开场冲击 / 沉闷僵持 / 温暖高点 / 收束品牌记忆 |
| ❺ 音效设计 | 环境音写具体拟声；`♪` 音乐仅一次成型模式写入 | 默认模式：提示词只写环境音与动作音效（"扑通水花声+卡通弹跳音效"），BGM 不写进视频提示词——视频模型自带的音乐后期换不掉；音乐设计留在脚本层，单独生成后合成时统一叠加。一次成型模式（用户要最快出片时）：♪ 音乐随镜头写入（"欢快的木琴+铃铛声突然弹入"），且合成阶段绝不再叠音乐 |
| ❻ 产品植入 | 在该镜头需要产品图入画的位置引用产品锚点图节点 | 让生成模型知道此刻展示的是哪张产品图、以什么状态入画 |
| ❼ 色温/氛围变化（可选） | 单独标注 | 仅在该镜头有色温或氛围转变时写，如"色调在冰箱门打开的瞬间从灰蓝转向暖黄" |

段内用 `｜` 分隔各要素，衔接用 `【】`，音乐用 `♪`。

## 声音叙事线

声音是一条从首镜贯到末镜的完整叙事线：环境音起 → 音乐渐入 → 高点 → 收束（末镜常"环境音：全静"，只留一声收尾音，如一声木琴 / 一记吉他和弦）。哪一镜音乐进、哪一镜到高点，必须和已确认脚本的情绪曲线对齐。这条线在**脚本层**完整设计；进视频提示词时按 ❺ 的模式取舍——默认只带环境音与音效，音乐线交给单独生成的 BGM 去实现。机位与节奏的对应：起用固定 / 缓慢运镜，转用最具冲击力的运镜，合回到固定镜头 + 渐显 slogan 文字。

## 关键约束（逐条自查）

1. 禁止抽象文案化：所有描述必须是镜头拍得到的，不写"象征着 / 代表了 / 传递出"。
2. 言简意赅交代故事，不堆比喻和情绪渲染的形容词——华丽的提示词小作文不会让画面更好，只会稀释有效信息；已上传主体 / 场景参考图的，文字不再重复描述外观，只写这一镜发生了什么。
3. 相邻镜头必有明确衔接逻辑；四种衔接类型轮换使用，不连续两镜用同一种。
4. 单镜时长标注是弱约束：模型实际按镜头数量分配时长，时间戳更多是给人看的节奏设计——想要节奏更紧凑，加镜头数（2-3 秒一镜），别指望改时间戳。
5. 产品植入克制：全片 2-3 次露出，首次不早于 SC2，最后一次落在收尾镜头。
6. 有 CG 转实拍、回忆转现实这类段落切换的方案，切换必须借一个共同元素无缝过渡（如 CG 果汁河涟漪 → 真实杯中液面涟漪），不许硬切。
7. 渐显文字（slogan / 品牌名）只用用户确认过的逐字口径，不自拟广告语。
8. 用户未要求字幕时，总控头末尾加反向约束"画面中不出现任何字幕、水印或多余文字"（用户确认的渐显 slogan 除外）——视频模型有凭空加字的习惯，必须显式防。
9. 一条提示词一整段连续文本输出，方便直接复制投喂。
